GCP Scenario-Based Interview Q&A
================================

BEGINNER LEVEL
--------------

Q1: You are asked to store raw CSV files uploaded daily by a business team. Which GCP service would you use and why?
A1: Use Cloud Storage. It is cost-effective, scalable, and supports object storage. Buckets can be organized by date or department, and lifecycle policies can manage retention.

Q2: A team wants to run SQL queries on large datasets without managing infrastructure. Which service fits best?
A2: BigQuery. It is serverless, supports ANSI SQL, and scales automatically for analytics workloads.

Q3: How would you give a user read-only access to a BigQuery dataset?
A3: Assign the IAM role `roles/bigquery.dataViewer` at the dataset level to the user.

INTERMEDIATE LEVEL
------------------

Q4: You need to process streaming data from IoT devices and store results for analytics. Which services would you use?
A4: Use Pub/Sub for ingestion, Dataflow for real-time processing, and BigQuery for analytics storage.

Q5: Your BigQuery queries are slow because of large scans. How do you optimize them?
A5: Use partitioned and clustered tables, avoid SELECT *, query only required columns, and use materialized views for repeated queries.

Q6: You are asked to orchestrate a pipeline that loads data from Cloud Storage → Dataflow → BigQuery. Which service helps manage this workflow?
A6: Cloud Composer. It uses Apache Airflow DAGs to schedule and monitor multi-step pipelines.

Q7: How would you enforce that analysts only see data for their region in BigQuery?
A7: Implement Row-level Security (RLS) policies that filter rows based on region attributes tied to user identity.

ADVANCED LEVEL
--------------

Q8: A company wants to migrate its on-prem Hadoop workloads to GCP. Which service would you recommend and why?
A8: Dataproc. It is a managed Hadoop/Spark service that allows easy migration of existing workloads with minimal code changes.

Q9: You need to design a data warehouse for analytics with historical tracking of dimension changes. Which modeling technique would you use?
A9: Dimensional modeling with Slowly Changing Dimensions (SCD). For example, SCD Type 2 to track historical changes by adding new rows.

Q10: How would you secure sensitive columns like salary in a BigQuery table while allowing analysts to query other fields?
A10: Use Column-level Security (CLS) to restrict access to sensitive columns. Analysts can query non-sensitive fields, while HR roles can access salary.

Q11: A pipeline must handle both batch and streaming data with minimal ops overhead. Which GCP service is best?
A11: Dataflow. It supports both batch and streaming via Apache Beam, is serverless, and scales automatically.

Q12: How do you ensure compliance and governance across multiple datasets in GCP?
A12: Use IAM roles for access control, Cloud Data Catalog for metadata management, audit logs for monitoring, and enforce encryption policies.

Q13: Your BigQuery costs are rising unexpectedly. What steps do you take?
A13: Review query patterns, optimize schema with partitioning/clustering, use cost controls (budgets/alerts), and educate teams to avoid inefficient queries.

Q14: You are asked to design a multi-tenant architecture where each client should only see their own data. How do you implement this in BigQuery?
A14: Use Row-level Security policies tied to tenant IDs, combined with IAM roles to restrict dataset access.

A15: Use BigQuery ML to train

Google Cloud Platform (GCP) Scenario-Based Interview Q&A
========================================================

BEGINNER LEVEL (Basics & Core Services)
---------------------------------------
Q1: You need to store images uploaded by users. Which GCP service do you use?
A1: Cloud Storage. It provides scalable object storage with buckets.

Q2: How do you give a user read-only access to a dataset in BigQuery?
A2: Assign IAM role `roles/bigquery.dataViewer` at the dataset level.

Q3: A team wants to run SQL queries without managing servers. Which service fits?
A3: BigQuery. It is serverless and supports ANSI SQL.

Q4: You want to schedule a daily job to load CSVs into BigQuery. Which service?
A4: Cloud Composer (Airflow DAG) or Cloud Scheduler with Dataflow.

Q5: How do you secure sensitive columns in BigQuery?
A5: Use Column-level Security (CLS) with IAM roles.

Q6: Which service is best for storing raw logs before processing?
A6: Cloud Storage or Pub/Sub (if streaming).

Q7: How do you enforce encryption in Cloud Storage?
A7: Use CMEK (Customer Managed Encryption Keys) or default encryption.

Q8: You need to share a dataset with another project. How?
A8: Grant IAM permissions at the dataset level to principals in the other project.

Q9: What’s the difference between primitive and predefined IAM roles?
A9: Primitive roles (Viewer, Editor, Owner) are broad; predefined are service-specific.

Q10: How do you monitor BigQuery query costs?
A10: Use GCP Billing reports, budgets, and query cost estimates.

INTERMEDIATE LEVEL (Data Pipelines & Optimization)
--------------------------------------------------
Q11: You need to process streaming IoT data. Which services do you use?
A11: Pub/Sub for ingestion, Dataflow for processing, BigQuery for analytics.

Q12: Your BigQuery queries are slow. How do you optimize?
A12: Use partitioned/clustering tables, avoid SELECT *, and materialized views.

Q13: How do you orchestrate a pipeline across multiple services?
A13: Use Cloud Composer with DAGs.

Q14: You need to migrate Hadoop jobs to GCP. Which service?
A14: Dataproc. Managed Hadoop/Spark clusters.

Q15: How do you enforce row-level restrictions in BigQuery?
A15: Use Row-level Security (RLS) policies tied to user attributes.

Q16: You want to automate ETL jobs triggered by file uploads. Which service?
A16: Cloud Functions + Pub/Sub + Dataflow.

Q17: How do you handle schema evolution in BigQuery?
A17: Use schema updates (add columns), or design flexible schemas with JSON.

Q18: You need to process batch data daily. Which service?
A18: Dataflow (batch pipelines) or Dataproc (batch jobs).

Q19: How do you integrate ML into BigQuery?
A19: Use BigQuery ML to train models directly with SQL.

Q20: How do you reduce Cloud Storage costs?
A20: Use lifecycle policies, Nearline/Coldline storage classes.

---------------------------------------------
Q21: You need multi-tenant architecture in BigQuery. How?
A21: Use RLS policies with tenant IDs and IAM dataset restrictions.

Q22: How do you ensure compliance with GDPR in GCP?
A22: Use IAM, encryption, audit logs, and Data Catalog for metadata governance.

Q23: Your Dataflow pipeline is failing under load. What do you do?
A23: Enable autoscaling, optimize transforms, and use windowing for streaming.

Q24: How do you design a warehouse with historical dimension tracking?
A24: Use Slowly Changing Dimensions (SCD Type 2).

Q25: How do you secure BigQuery datasets across multiple teams?
A25: Use IAM roles, authorized views, RLS, and CLS.

Q26: You need to integrate external data sources into BigQuery. How?
A26: Use federated queries (Cloud Storage, Sheets, Bigtable).

Q27: How do you monitor pipeline health in Composer?
A27: Use Airflow DAG monitoring, Cloud Logging, and alerts.

Q28: You want to train ML models on streaming data. Which service?
A28: Dataflow + TensorFlow or BigQuery ML for batch ML.

Q29: How do you enforce least privilege in IAM?
A29: Assign predefined roles, avoid primitive roles, use service accounts.

Q30: You need disaster recovery for BigQuery. How?
A30: Replicate datasets across regions, export to Cloud Storage.
